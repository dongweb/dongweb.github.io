hexo搭建静态博客步骤

发表于 2014-09-18   |   分类于 关于博客   |  

如何用hexo搭建自己的静态博客

既然用了hexo搭建了博客，当然得把自己的过程记录一下，无论是对新事物的巩固还是经验的分享都需要如此：

准备:

一：本地环境

安装node和git，具体安装方法google或者baidu去吧.

二：github

github的注册以及使用我也不再重述，网上很多，我也不想复制别人的东西，完全是多余。你可以查找相关资料。

注册完成后我们要用到的功能就是它的github pages，也就是个人主页仓库，github每个账户只能有一个仓库用来存放个人主页，仓库的名字必须是username/username.github.io，这是github规定死的就这么写；目的是方便你通过http://username.github.io这种形式来访问你的个人主页。

如果上面的看起来有点困难，好的，那么你只需要做以下几步：

  1. 打开github首页,点击右下方的那个绿色按钮(+New repository).
  2. 在新打开的页面Repository name的位置输入你的个人主页仓库名,比如你的github账户名是zhangsan,那么此处就填写zhangsan.github.io,填好后直接点下方的绿色按钮(Creating repository),其他不需要做任何操作。
  3. 完成第二步github会到一个仓库的简单信息的页面，这个页面的主要是用来告诉你如何通过git来关联远程仓库的，也就是你的zhangsan.github.io这个仓库。完成这步以后我们就该在本地环境下搭建博客了.

我用的mac,如果你是windows系统可能不需要加前面的sudo.

四：配置

在根目录下找到_config.yml这个文件，里面就是配置信息，这步可以网上搜索一下，基本都是一样的。

五：调试

首先发表一篇文章试一下。在命令行执行如下命令：

hexo new 'A Test Article'

完了以后你会发现在source/_posts这个目录下会有你刚创建的名为“A-Test-Article.md”的文章，你所要写的文章内容就是在这个文件里操作的，可以看下我就是这么编写这篇文章的：

编辑完以后重启本地服务hexo server，会发现新的文章已经有了。

六：选择博客主题

如果你用默认的主题也可以，如果想选择一个其他主题可以google搜索一下”hexo 主题”,当然我使用的主题是某某人自己搞的主题，然后他把主题放到了github上，这样大家都可以使用他的主题了，也算是现代程序员的一种开源精神，github本身就开源。

我们以pacman这个主题为例子，为什么用他的，因为我也是参考别人文档直接用了他的。哈哈，废话不多说，步骤如下:

执行命令：

git clone https://github.com/A-limon/pacman.git themes/pacman

完了以后在根目录编辑配置文件_config.yml,找到theme字段,将值改为pacman,注意值的前面是有个空格的，hexo配置文件里的值都是如此。重启服务，刷新本地页面，会发现主题已经变为pacman了。

七：部署到github

在部署之前我们要配置一下_config.yml。在配置之前首先执行一下这个命令：

npm install hexo-deployer-git --save

九：补充

到现在我们已经用hexo顺利的部署了一个静态博客了，当然虽说静态，但也算是一高逼格的东西。剩下的是对上述的一个补充。
